# HTB Academy Write-up: cURL GET Requests and Basic Authentication

---

# Objective

Learn how to use **cURL** to interact with web applications, authenticate using **HTTP Basic Authentication**, inspect HTTP headers, and perform authenticated GET requests.

---

# HTTP Basic Authentication

Basic Authentication is an HTTP authentication mechanism where the client sends the username and password with every request.

When authentication is required, the server responds with:

```
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm="Access denied"
```

The browser or client then sends an **Authorization** header containing the Base64 encoded credentials.

---

# Accessing Protected Resources

### Using Username and Password

```bash
curl -u admin:admin http://SERVER_IP
```

Equivalent URL format:

```bash
curl http://admin:admin@SERVER_IP
```

Both methods authenticate using HTTP Basic Authentication.

---

# Viewing HTTP Headers

Use the **-i** option to display response headers.

```bash
curl -i http://SERVER_IP
```

Unauthenticated requests usually return:

```
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Basic realm="Access denied"
```

Authenticated requests return:

```
HTTP/1.1 200 OK
```

---

# Verbose Requests

Use **-v** to inspect the complete HTTP request and response.

```bash
curl -v -u admin:admin http://SERVER_IP
```

Example request:

```
Authorization: Basic YWRtaW46YWRtaW4=
```

The value is simply:

```
Base64(admin:admin)
```

---

# Supplying the Authorization Header

Instead of using **-u**, the Authorization header can be supplied manually.

```bash
curl -H "Authorization: Basic YWRtaW46YWRtaW4=" http://SERVER_IP
```

This produces the same authenticated request.

---

# GET Parameters

GET requests pass user input through the URL.

Example:

```
search.php?search=london
```

Parameter:

| Parameter | Value |
|-----------|--------|
| search | london |

---

# Sending GET Requests with cURL

```bash
curl "http://SERVER_IP/search.php?search=london" \
-H "Authorization: Basic YWRtaW46YWRtaW4="
```

Example response:

```
London (UK)
```

---

# Browser Developer Tools

The Network tab allows you to inspect requests generated by the browser.

Useful actions include:

- View Request URL
- View HTTP Method
- Inspect Headers
- View Query Parameters
- Copy as cURL
- Copy as Fetch

These options make it easy to reproduce browser requests from the terminal.

---

# Copy as cURL

Modern browsers can automatically generate the equivalent cURL command.

Example:

```
Right Click Request
    → Copy
        → Copy as cURL
```

This saves time when reproducing authenticated requests.

---

# Key Takeaways

- HTTP Basic Authentication uses the Authorization header.
- Credentials are Base64 encoded, not encrypted.
- GET parameters are transmitted in the URL.
- cURL can fully reproduce browser requests.
- Browser DevTools simplify request analysis by providing "Copy as cURL" and "Copy as Fetch".
