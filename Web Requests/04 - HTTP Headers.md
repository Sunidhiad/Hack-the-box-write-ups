# HTB Academy Write-up: HTTP Headers

## Objective

Understand the purpose of HTTP headers, identify common request and response headers, and use browser Developer Tools and cURL to inspect HTTP communications.

---

# What are HTTP Headers?

HTTP headers are key-value pairs exchanged between the client and the server that provide additional information about the HTTP request or response.

Example:

```http
Host: example.com
User-Agent: Mozilla/5.0
Content-Type: text/html
```

Headers help define how requests should be processed and how responses should be interpreted.

---

# Types of HTTP Headers

## 1. General Headers

Used in both requests and responses.

Examples:

* Date
* Connection

---

## 2. Entity Headers

Describe the content being transferred.

Examples:

* Content-Type
* Content-Length
* Content-Encoding
* Media-Type
* Boundary

---

## 3. Request Headers

Sent by the client to the server.

Common request headers:

| Header        | Purpose                               |
| ------------- | ------------------------------------- |
| Host          | Specifies the target host             |
| User-Agent    | Identifies the client                 |
| Accept        | Specifies acceptable response formats |
| Cookie        | Sends stored session cookies          |
| Referer       | Indicates the previous page           |
| Authorization | Sends authentication credentials      |

---

## 4. Response Headers

Returned by the server.

Common response headers:

| Header           | Purpose                               |
| ---------------- | ------------------------------------- |
| Server           | Server software information           |
| Set-Cookie       | Creates cookies on the client         |
| WWW-Authenticate | Indicates authentication requirements |

---

## 5. Security Headers

Improve browser security.

Examples:

* Content-Security-Policy (CSP)
* Strict-Transport-Security (HSTS)
* Referrer-Policy

---

# Useful cURL Commands

View only response headers:

```bash
curl -I http://example.com
```

View headers and response body:

```bash
curl -i http://example.com
```

Verbose request and response:

```bash
curl -v http://example.com
```

Change the User-Agent:

```bash
curl -A "Mozilla/5.0" http://example.com
```

---

# Browser Developer Tools

The browser Network tab allows us to:

* View every HTTP request
* Inspect request headers
* Inspect response headers
* View cookies
* View request timing
* View the response body

This is extremely useful during web application penetration testing.

# HTB Exercise

## Objective

Use the browser's Developer Tools (Network tab) to identify the request made after the page loads and retrieve the flag from its response.

### Steps

1. Open the target website.
2. Press **F12** or **Ctrl + Shift +I** to open Developer Tools.
3. Navigate to the **Network** tab.
4. Refresh the page.
5. Observe all HTTP requests generated by the page.
6. Locate the request made to:

```text
flag_327a6c4304ad5938eaf0efb6cc3e53dc.txt
```

7. Click on the request.
8. Open the **Response** tab.
9. Read the contents of the response body to obtain the flag.

---

## Flag

```text
HTB{p493_r3qu3$t$_m0n!t0r}
```

---

# Key Takeaways

* HTTP headers provide metadata about requests and responses.
* Request headers are sent by the client.
* Response headers are returned by the server.
* Security headers help mitigate common web attacks.
* `curl -I` displays only response headers.
* `curl -i` displays both response headers and the response body.
* Browser Developer Tools are essential for monitoring web traffic.
* The Network tab can reveal additional HTTP requests made after the initial page load.
* The actual flag was stored in the **response body** of a secondary request, not in the filename itself.
